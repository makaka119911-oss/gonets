# Модель данных «Гонец» (универсальная доставка)

## Роли (`User.role`)

| Prisma enum | Описание |
|-------------|----------|
| `CLIENT` | Клиент (заказы как получатель/отправитель) |
| `COURIER` | Курьер |
| `SELLER` | Магазин / продавец (заказы с привязкой к магазину) |
| `ADMIN` | Администратор |

В API и моках в JSON роли в **нижнем регистре** (`client`, `courier`, …).

## Категории отправления (`Order.shipmentCategory`)

| Enum | Смысл |
|------|--------|
| `FOOD` | Еда |
| `PARCEL` | Посылки |
| `DOCUMENTS` | Документы |
| `GOODS` | Товары |
| `BULKY` | Крупногабарит |

## Статусы заказа (`Order.status`)

Жизненный цикл (упрощённо): создание → назначение курьера → забор → в пути → у получателя → доставлено / отмена / сбой.

| Enum | Подпись в API (`statusLabel`) |
|------|-------------------------------|
| `CREATED` | Создан |
| `CONFIRMED` | Подтверждён |
| `COURIER_ASSIGNED` | Курьер назначен |
| `COURIER_ON_WAY` | Курьер в пути |
| `PICKED_UP` | Забрано |
| `IN_TRANSIT` | В пути |
| `AT_RECIPIENT` | У получателя |
| `DELIVERED` | Доставлено |
| `CANCELED` | Отменено |
| `FAILED` | Сбой |

В ответах REST поле `status` — **snake_case** (`courier_on_way`, `delivered`, …) для совместимости со статическим сайтом.

## Трекинг

События хранятся в `OrderEvent` (связь `orderId`, поле `title`, `createdAt`). В ответе `GET /api/orders/:id` массив `events` с полями `time`, `title` (время форматируется на сервере).

## Связи заказа

- `clientId` — всегда клиент.
- `courierId` — назначенный курьер (опционально).
- `sellerId` — магазин-источник (опционально, для заказов от продавца).

Фильтрация списка заказов по роли реализована в `OrdersService.listForUser`.
