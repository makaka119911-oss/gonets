# MVP Scope — Гонец (Москва + МО)

## Цель релиза

Запустить коммерчески пригодный MVP для первых платящих клиентов:
- физлица: разовые отправки;
- юрлица: базовый поток регулярных доставок.

## Must-have (входит в MVP)

## 1) Клиент (веб + мобильный клиент)
- Расчет стоимости доставки.
- Создание заказа с адресами, весом, типом груза, временем.
- Загрузка фото груза (базово).
- Оплата онлайн (минимум 1 провайдер + СБП по возможности).
- Отслеживание статуса заказа.
- История заказов и повтор маршрута.

## 2) Курьер (мобильный)
- Регистрация и базовая верификация.
- Просмотр доступных заказов рядом.
- Принятие/отказ.
- Смена статусов (в пути, забрал, доставлено).
- Фото подтверждение + код/подпись получения.
- Баланс и запрос на вывод.

## 3) Админка
- Лента всех заказов, фильтры, ручное назначение.
- Управление курьерами (подтверждение/блокировка).
- Управление тарифами.
- Просмотр выплат и ручные корректировки.
- Жалобы/инциденты в базовом виде.

## 4) Платформа
- Авторизация (JWT + refresh).
- Push/SMS/email уведомления.
- Роли и права доступа.
- Базовая аналитика по заказам.

## Out of scope (фаза 2)

- Полноценный овердрафт.
- Глубокая CRM-интеграция (amoCRM/RetailCRM).
- Сложные маршруты multi-drop.
- Расширенная BI-панель.
- Гибкая промо-механика и реферальные деревья.
- Продвинутый антифрод и ML-предикты ETA.

## KPI MVP

- Среднее время назначения курьера: до 7 минут.
- Доля успешно завершенных заказов: >= 95%.
- NPS клиентов: >= 45.
- Время отклика API p95: < 600 мс.
