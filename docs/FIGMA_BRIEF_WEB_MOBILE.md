# Figma Brief — единый стиль Web + Mobile (Клиент)

## Цель

Собрать целостный UX, где сайт и приложение клиента выглядят как один продукт.

## Deliverables

1. UI Kit (общая библиотека):
- color tokens;
- typography scale;
- spacing;
- buttons, inputs, chips, status badges;
- cards, modals, bottom sheets;
- nav patterns (web header, mobile tab bar).

2. Web screens:
- Landing + calculator;
- Create Order (multi-step);
- Checkout;
- Tracking;
- Order history;
- Login/register modals.

3. Mobile client screens:
- Onboarding/login;
- Quick calculate;
- Create order;
- Payment;
- Active order tracking;
- Chat;
- History/reorder;
- Profile/promocodes.

4. Prototype flows:
- "Рассчитать -> Оформить -> Оплатить -> Трекинг";
- "История -> Повторить заказ".

## Базовые UX-правила

- Одни и те же названия статусов на вебе и мобайле.
- Одни и те же цвета статусов.
- CTA одного смысла одного цвета во всех поверхностях.
- Ошибки полей и платежа — единый паттерн.

## Состояния для отрисовки (обязательно)

- loading/skeleton;
- empty state;
- offline/no internet;
- validation error;
- payment fail;
- order cancelled;
- delivered success.

## Визуальные ориентиры

- Минимализм, чистые карточки, читаемый контраст.
- Акцентный цвет бренда "Гонец" (розовый/магентовый диапазон).
- Светлая и темная тема на базе одних токенов.

## Ограничения MVP

- Не проектировать "идеальный world-class superapp" сразу.
- Только экраны, которые реально входят в релиз MVP.
